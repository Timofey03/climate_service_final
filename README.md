# Climate Service — учёт заявок на ремонт климатического оборудования

## Что внутри
- `table.sql` — схема БД (PostgreSQL)
- `database_module.py` — слой доступа к данным (DAO)
- `main_app.py` — GUI (PyQt6): авторизация, заявки, комментарии, статистика, QR-код
- `import_data.py` — импорт данных из файлов `inputData*.csv`
- `qr_generator.py` — генератор QR-кода на форму оценки качества
- `test_system.py` — примеры функциональных тестов (на основные функции)

## Быстрый старт (Windows)
1. Установите PostgreSQL и создайте БД (например `climate_service`)
2. Выполните SQL:
   - откройте `table.sql` и выполните в вашей БД
3. Настройте переменные окружения (или поправьте параметры в `Database(...)`):
   - `DB_HOST`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`, `DB_PORT`
4. Импортируйте данные:
   ```bash
   python import_data.py
   ```
5. Запустите приложение:
   ```bash
   python main_app.py
   ```

## Роли
- Оператор — регистрация/редактирование заявок, назначение специалиста
- Специалист — комментарии, комплектующие, изменение статуса по работам
- Менеджер по качеству — консультации, продление сроков (через поле "Срок/Completion date"), привлечение специалистов (переназначение/комментарий)

По умолчанию создаётся пользователь `admin/admin` (роль Оператор), если его нет в БД.
